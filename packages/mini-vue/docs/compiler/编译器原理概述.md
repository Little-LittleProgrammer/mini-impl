# 编译器原理概述

## 什么是编译器？

编译器是将一种语言的代码转换为另一种语言的程序。在 Vue 中，编译器负责将**模板字符串**转换为**渲染函数**。

```typescript
// 输入：模板字符串
const template = `
  <div class="app">
    <h1>{{ title }}</h1>
    <p>Count: {{ count }}</p>
  </div>
`

// 输出：渲染函数
function render(_ctx, _cache) {
  return createElementVNode("div", { class: "app" }, [
    createElementVNode("h1", null, toDisplayString(_ctx.title)),
    createElementVNode("p", null, "Count: " + toDisplayString(_ctx.count))
  ])
}
```

## 编译流程

Vue 编译器采用三阶段架构：

```
┌─────────────────────────────────────────────────────────────────┐
│                        编译流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Template String                                                 │
│       │                                                          │
│       ▼                                                          │
│  ┌─────────────┐                                                 │
│  │    Parse    │  解析模板，生成 AST                              │
│  └─────────────┘                                                 │
│       │                                                          │
│       ▼                                                          │
│       AST（抽象语法树）                                            │
│       │                                                          │
│       ▼                                                          │
│  ┌─────────────┐                                                 │
│  │  Transform  │  转换 AST，生成 JavaScript AST                   │
│  └─────────────┘                                                 │
│       │                                                          │
│       ▼                                                          │
│  JavaScript AST                                                  │
│       │                                                          │
│       ▼                                                          │
│  ┌─────────────┐                                                 │
│  │  Generate   │  生成渲染函数代码                                │
│  └─────────────┘                                                 │
│       │                                                          │
│       ▼                                                          │
│  Render Function Code                                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 编译入口

```typescript
// packages/compiler-core/src/compile.ts

export function baseCompile(
  template: string,
  options: CompilerOptions = {}
): CodegenResult {
  // 1. 解析模板，生成 AST
  const ast = parse(template, options)

  // 2. 转换 AST
  transform(ast, {
    ...options,
    nodeTransforms: [
      transformElement,
      transformText,
      ...options.nodeTransforms || []
    ]
  })

  // 3. 生成代码
  return generate(ast, options)
}
```

## 阶段一：Parse（解析）

解析器将模板字符串转换为 AST（抽象语法树）。

### AST 节点类型

```typescript
// packages/compiler-core/src/ast.ts

export const enum NodeTypes {
  // 原始节点类型
  ROOT,              // 根节点
  ELEMENT,           // 元素节点 <div>
  TEXT,              // 文本节点 "hello"
  COMMENT,           // 注释节点 <!-- comment -->
  INTERPOLATION,     // 插值表达式 {{ msg }}
  SIMPLE_EXPRESSION, // 简单表达式
  ATTRIBUTE,         // 普通属性 class="app"
  DIRECTIVE,         // 指令节点 v-if, v-for

  // 转换后的 JavaScript AST 节点
  VNODE_CALL,        // createElementVNode 调用
  JS_CALL_EXPRESSION,    // 函数调用
  JS_OBJECT_EXPRESSION,  // 对象字面量
  JS_PROPERTY,           // 对象属性
  JS_ARRAY_EXPRESSION,   // 数组字面量
  JS_FUNCTION_EXPRESSION, // 函数表达式
  JS_CONDITIONAL_EXPRESSION, // 三元表达式
  JS_CACHE_EXPRESSION    // 缓存表达式
}
```

### 示例：模板到 AST

```typescript
// 输入模板
const template = `<div class="app">{{ msg }}</div>`

// 输出 AST
const ast = {
  type: NodeTypes.ROOT,
  children: [
    {
      type: NodeTypes.ELEMENT,
      tag: 'div',
      props: [
        {
          type: NodeTypes.ATTRIBUTE,
          name: 'class',
          value: {
            type: NodeTypes.TEXT,
            content: 'app'
          }
        }
      ],
      children: [
        {
          type: NodeTypes.INTERPOLATION,
          content: {
            type: NodeTypes.SIMPLE_EXPRESSION,
            content: 'msg',
            isStatic: false
          }
        }
      ]
    }
  ]
}
```

详细内容请参阅 [Parser 解析器原理](./Parser解析器原理.md)。

## 阶段二：Transform（转换）

转换器将 AST 转换为 JavaScript AST，并应用各种优化。

### 转换过程

```typescript
// packages/compiler-core/src/transform.ts

export function transform(root: RootNode, options: TransformOptions) {
  // 创建转换上下文
  const context = createTransformContext(root, options)

  // 遍历 AST，应用转换
  traverseNode(root, context)

  // 创建根节点的 codegenNode
  createRootCodegen(root)

  // 收集使用的帮助函数
  root.helpers = [...context.helpers.keys()]
}
```

### 转换插件

转换器使用插件机制，每个插件处理特定类型的节点：

```typescript
// 元素转换
export const transformElement = (node, context) => {
  // 将元素节点转换为 VNodeCall
  node.codegenNode = createVNodeCall(context, tag, props, children)
}

// 文本转换
export const transformText = (node, context) => {
  // 合并相邻的文本和插值表达式
}

// v-if 转换
export const transformIf = (node, context) => {
  // 将 v-if 转换为条件表达式
}
```

### 深度优先遍历

转换器采用深度优先遍历，分两个阶段：

```
┌─────────────────────────────────────────────────────────────────┐
│                    遍历过程                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  进入节点 → 执行转换插件 → 收集退出函数                           │
│      │                                                           │
│      ▼                                                           │
│  递归处理子节点                                                   │
│      │                                                           │
│      ▼                                                           │
│  退出节点 → 倒序执行退出函数                                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

详细内容请参阅 [Transformer 转换器原理](./Transformer转换器原理.md)。

## 阶段三：Generate（生成）

代码生成器将 JavaScript AST 转换为可执行的渲染函数代码。

### 生成的代码结构

```typescript
// 生成的代码结构
const code = `
  const { createElementVNode: _createElementVNode, toDisplayString: _toDisplayString } = _Vue

  return function render(_ctx, _cache) {
    return _createElementVNode("div", { class: "app" }, [
      _createElementVNode("h1", null, _toDisplayString(_ctx.title))
    ])
  }
`
```

### 代码生成过程

```typescript
// packages/compiler-core/src/codegen.ts

export function generate(ast: RootNode, options: CodegenOptions): CodegenResult {
  // 创建代码生成上下文
  const context = createCodegenContext(ast, options)

  // 生成前置代码（导入帮助函数）
  genFunctionPreamble(context)

  // 生成渲染函数
  push(`function render(_ctx, _cache) {`)
  indent()
  push(`with (_ctx) {`)
  indent()

  // 生成帮助函数解构
  push(`const { ${ast.helpers.map(aliasHelper).join(', ')} } = _Vue`)

  // 生成 return 语句
  push(`return `)
  genNode(ast.codegenNode, context)

  deindent()
  push(`}`)
  deindent()
  push(`}`)

  return {
    ast,
    code: context.code
  }
}
```

详细内容请参阅 [Codegen 代码生成器原理](./Codegen代码生成器原理.md)。

## 运行时帮助函数

编译器生成的代码会使用运行时帮助函数，这些函数在运行时提供。

### 常用帮助函数

```typescript
// packages/compiler-core/src/runtimeHelpers.ts

export const enum RuntimeHelperNames {
  // VNode 创建
  CREATE_ELEMENT_VNODE = 'createElementVNode',
  CREATE_TEXT_VNODE = 'createTextVNode',
  CREATE_COMMENT_VNODE = 'createCommentVNode',

  // 显示
  TO_DISPLAY_STRING = 'toDisplayString',

  // 指令
  RENDER_LIST = 'renderList',
  WITH_DIRECTIVES = 'withDirectives',

  // 缓存
  CACHE = 'cache',
  CACHE_HANDLER = 'cacheHandler'
}

// Symbol 形式
export const CREATE_ELEMENT_VNODE = Symbol(`createElementVNode`)
export const TO_DISPLAY_STRING = Symbol(`toDisplayString`)
```

### 帮助函数收集

在转换过程中，当需要使用帮助函数时，将其添加到 `context.helpers`：

```typescript
// 使用帮助函数
context.helper(TO_DISPLAY_STRING)
context.helper(CREATE_ELEMENT_VNODE)

// 收集到 AST
root.helpers = [...context.helpers.keys()]
// [Symbol(toDisplayString), Symbol(createElementVNode)]
```

### 代码生成

```typescript
// 生成的代码
const { toDisplayString: _toDisplayString, createElementVNode: _createElementVNode } = _Vue

return function render(_ctx, _cache) {
  return _createElementVNode("div", null, _toDisplayString(_ctx.msg))
}
```

## 静态提升

静态提升是一种优化策略，将静态节点提升到渲染函数外部，避免每次渲染都重新创建。

### 示例

```typescript
// 输入模板
const template = `
  <div>
    <span class="static">静态内容</span>
    <span>{{ dynamic }}</span>
  </div>
`

// 不优化：每次渲染都创建静态节点
function render(_ctx) {
  return createElementVNode("div", null, [
    createElementVNode("span", { class: "static" }, "静态内容"),
    createElementVNode("span", null, toDisplayString(_ctx.dynamic))
  ])
}

// 静态提升：静态节点只创建一次
const _hoisted_1 = createElementVNode("span", { class: "static" }, "静态内容", -1)

function render(_ctx) {
  return createElementVNode("div", null, [
    _hoisted_1,
    createElementVNode("span", null, toDisplayString(_ctx.dynamic))
  ])
}
```

## 编译器核心设计原则

### 1. 平台无关

`compiler-core` 不依赖任何平台特定的 API，所有平台相关的逻辑在 `compiler-dom` 中处理。

```typescript
// compiler-dom/src/index.ts
export function compile(template: string, options = {}) {
  return baseCompile(template, {
    ...options,
    // 浏览器特定的配置
    isNativeTag: tag => isHTMLTag(tag),
    isBuiltInComponent: tag => isBuiltIn(tag)
  })
}
```

### 2. 插件化

转换器采用插件机制，可以轻松扩展。

```typescript
// 自定义转换插件
const myTransform = (node, context) => {
  if (node.type === NodeTypes.ELEMENT) {
    // 自定义处理
  }
}

transform(ast, {
  nodeTransforms: [
    transformElement,
    transformText,
    myTransform  // 添加自定义插件
  ]
})
```

### 3. 可优化

编译器会识别可优化的模式：

- 静态提升
- 静态节点标记（PatchFlags）
- 缓存事件处理器

## 编译模式

### 运行时编译

在浏览器中实时编译模板。

```typescript
import { compile } from 'vue'

const { render } = compile('<div>{{ msg }}</div>')
```

特点：
- 灵活，可以使用动态模板
- 需要携带编译器代码
- 编译有性能开销

### 预编译

在构建时编译模板（推荐）。

```typescript
// .vue 文件
<template>
  <div>{{ msg }}</div>
</template>

// 编译后
import { render } from './App.vue'
```

特点：
- 更小的打包体积（不需要编译器）
- 更快的启动速度
- 更好的类型支持

## Source Map

编译器支持生成 Source Map，便于调试。

```typescript
const { code, map } = compile(template, {
  sourceMap: true
})
```

## 总结

Vue 编译器的核心流程：

| 阶段 | 输入 | 输出 | 功能 |
|------|------|------|------|
| Parse | 模板字符串 | AST | 解析模板结构 |
| Transform | AST | JavaScript AST | 转换和优化 |
| Generate | JavaScript AST | 渲染函数代码 | 生成可执行代码 |
