# mini-vue 总览

## 项目简介

mini-vue 是 Vue 3 核心功能的简化实现，旨在帮助开发者深入理解 Vue 的底层原理。通过学习 mini-vue，你将掌握：

- **响应式系统**：如何实现数据驱动视图更新
- **编译器**：如何将模板转换为渲染函数
- **运行时**：如何高效地渲染和更新 DOM
- **组件系统**：组件的创建、挂载和更新流程

## 架构设计

mini-vue 采用 **Monorepo** 架构，将不同功能拆分为独立的包：

```
packages/mini-vue/
├── packages/
│   ├── reactivity/       # 响应式系统
│   ├── compiler-core/    # 编译器核心（平台无关）
│   ├── compiler-dom/     # 浏览器特定编译
│   ├── runtime-core/     # 运行时核心（平台无关）
│   ├── runtime-dom/      # 浏览器 DOM 操作
│   ├── shared/           # 共享工具函数
│   └── vue/              # 入口，整合所有模块
```

## 核心概念

### 1. 响应式系统 (Reactivity)

响应式系统是 Vue 的核心，它实现了数据变化自动更新视图的能力。

```typescript
import { reactive, effect } from 'vue'

// 创建响应式对象
const state = reactive({ count: 0 })

// 副作用函数：当 state.count 变化时自动重新执行
effect(() => {
  console.log('count is:', state.count)
})

// 修改数据，触发副作用重新执行
state.count++  // 控制台输出: count is: 1
```

**核心机制**：
- **依赖收集**：在副作用执行时，记录它访问了哪些响应式数据
- **触发更新**：当响应式数据变化时，通知所有依赖它的副作用重新执行

### 2. 编译器 (Compiler)

编译器将模板字符串转换为渲染函数。

```typescript
import { compile } from 'vue'

const template = `
  <div class="app">
    <h1>{{ title }}</h1>
    <p>Count: {{ count }}</p>
  </div>
`

const { code } = compile(template)
// 生成的渲染函数代码：
// function render(_ctx, _cache) {
//   return createElementVNode("div", { class: "app" }, [
//     createElementVNode("h1", null, toDisplayString(_ctx.title)),
//     createElementVNode("p", null, "Count: " + toDisplayString(_ctx.count))
//   ])
// }
```

**编译流程**：
```
Template String → Parse → AST → Transform → JavaScript AST → Generate → Render Function
```

### 3. 运行时 (Runtime)

运行时负责将渲染函数的执行结果渲染到真实 DOM。

```typescript
import { createApp, h } from 'vue'

const App = {
  setup() {
    const state = reactive({ count: 0 })
    return () => h('div', null, state.count)
  }
}

createApp(App).mount('#app')
```

**核心流程**：
1. **创建组件实例**：初始化组件状态
2. **执行 setup**：获取响应式数据和渲染函数
3. **执行渲染函数**：生成虚拟 DOM
4. **渲染到 DOM**：将虚拟 DOM 转换为真实 DOM
5. **响应式更新**：数据变化时重新渲染

## 设计理念

### 1. 平台无关性

编译器和运行时的核心逻辑与平台无关：
- `compiler-core` 不依赖浏览器 API
- `runtime-core` 通过 `RendererOptions` 接口抽象 DOM 操作

这使得 Vue 可以轻松适配不同平台（浏览器、小程序、原生应用等）。

### 2. 模块化

每个包职责单一，可以独立使用：
- 可以单独使用 `reactivity` 包实现响应式
- 可以单独使用 `compiler-core` 进行模板编译

### 3. Tree-shaking 友好

通过 ES Module 导出，未使用的代码可以在打包时被移除。

## 与 Vue 3 的差异

mini-vue 是 Vue 3 的简化实现，主要差异：

1. **功能精简**：只实现核心功能，省略了 SSR、Teleport、Suspense 等
2. **错误处理简化**：生产环境级别的错误处理和边界情况处理被简化
3. **性能优化有限**：省略了部分性能优化策略
4. **类型定义简化**：使用更简单的类型定义

但核心原理与 Vue 3 完全一致，学习 mini-vue 可以直接迁移到 Vue 3 的理解。
