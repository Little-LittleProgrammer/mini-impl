# 响应式系统概述

## 什么是响应式？

响应式是一种编程范式，当数据发生变化时，依赖该数据的代码会自动重新执行。在 Vue 中，这意味着当数据变化时，视图会自动更新。

```typescript
// 非响应式：数据变化，视图不会更新
let count = 0
document.getElementById('app').textContent = count
count = 1  // 视图仍然显示 0

// 响应式：数据变化，视图自动更新
const state = reactive({ count: 0 })
effect(() => {
  document.getElementById('app').textContent = state.count
})
state.count = 1  // 视图自动更新为 1
```

## 响应式系统的核心概念

### 1. 副作用 (Effect)

副作用是指会产生外部影响的函数。在 Vue 中，渲染函数就是一个副作用，因为它会修改 DOM。

```typescript
// 这是一个副作用函数
function sideEffect() {
  document.body.textContent = 'Hello'
}
```

### 2. 依赖收集 (Dependency Tracking)

当副作用执行时，记录它访问了哪些响应式数据。这些数据就是该副作用的"依赖"。

```typescript
const state = reactive({ count: 0, name: 'Vue' })

effect(() => {
  // 这个副作用依赖 state.count，但不依赖 state.name
  console.log(state.count)
})
```

### 3. 触发更新 (Trigger)

当响应式数据变化时，通知所有依赖它的副作用重新执行。

```typescript
state.count++  // 触发上面的 effect 重新执行
```

## 响应式系统的实现原理

### 核心数据结构

```
┌─────────────────────────────────────────────────────────────────┐
│                         targetMap                                │
│  (WeakMap: target → KeyToDepMap)                                │
├─────────────────────────────────────────────────────────────────┤
│  target (响应式对象)                                              │
│    │                                                             │
│    └── KeyToDepMap (Map: key → Dep)                             │
│          │                                                       │
│          ├── "count" → Dep (Set<ReactiveEffect>)                │
│          │                ├── effect1                           │
│          │                └── effect2                           │
│          │                                                       │
│          └── "name" → Dep (Set<ReactiveEffect>)                 │
│                         └── effect3                              │
└─────────────────────────────────────────────────────────────────┘
```

### 工作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        依赖收集阶段                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. effect(fn) 执行                                              │
│       │                                                          │
│       ▼                                                          │
│  2. activeEffect = 当前 effect                                   │
│       │                                                          │
│       ▼                                                          │
│  3. 执行 fn()                                                    │
│       │                                                          │
│       ▼                                                          │
│  4. fn() 中访问 state.count                                      │
│       │                                                          │
│       ▼                                                          │
│  5. Proxy get 拦截器触发                                         │
│       │                                                          │
│       ▼                                                          │
│  6. track(target, 'count')                                       │
│       │                                                          │
│       ▼                                                          │
│  7. 将 activeEffect 添加到 targetMap[target]['count']            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                        触发更新阶段                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. state.count = newValue                                      │
│       │                                                          │
│       ▼                                                          │
│  2. Proxy set 拦截器触发                                         │
│       │                                                          │
│       ▼                                                          │
│  3. trigger(target, 'count')                                     │
│       │                                                          │
│       ▼                                                          │
│  4. 从 targetMap 获取所有依赖 'count' 的 effects                │
│       │                                                          │
│       ▼                                                          │
│  5. 执行所有 effects                                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## mini-vue 响应式 API

### reactive()

创建响应式对象，适用于对象类型。

```typescript
import { reactive } from 'vue'

const state = reactive({
  count: 0,
  nested: {
    value: 'hello'
  }
})

// 深层响应式
state.nested.value = 'world'  // 触发更新
```

### ref()

创建响应式引用，适用于基本类型或需要替换整个对象的场景。

```typescript
import { ref } from 'vue'

const count = ref(0)
const obj = ref({ name: 'Vue' })

// 通过 .value 访问和修改
console.log(count.value)  // 0
count.value++             // 触发更新
```

### computed()

创建计算属性，具有缓存和懒计算特性。

```typescript
import { ref, computed } from 'vue'

const count = ref(0)
const doubled = computed(() => count.value * 2)

console.log(doubled.value)  // 0
count.value = 1
console.log(doubled.value)  // 2 (重新计算)
```

### watch()

监听响应式数据的变化。

```typescript
import { ref, watch } from 'vue'

const count = ref(0)

watch(count, (newValue, oldValue) => {
  console.log(`count changed from ${oldValue} to ${newValue}`)
})

count.value++  // 控制台输出: count changed from 0 to 1
```

### effect()

底层 API，用于创建副作用。

```typescript
import { reactive, effect } from 'vue'

const state = reactive({ count: 0 })

effect(() => {
  console.log('count is:', state.count)
})

state.count++  // 控制台输出: count is: 1
```

## 响应式系统的设计目标

### 1. 自动依赖收集

开发者不需要手动声明依赖，系统自动追踪。

```typescript
// 不需要这样
watch(['state.count', 'state.name'], () => { ... })

// 只需要这样
effect(() => {
  // 自动收集 state.count 和 state.name 作为依赖
  console.log(state.count, state.name)
})
```

### 2. 精确更新

只有真正使用到的属性变化才会触发更新。

```typescript
const state = reactive({ count: 0, name: 'Vue' })

effect(() => {
  // 只依赖 count
  console.log(state.count)
})

state.name = 'React'  // 不会触发 effect
state.count++         // 触发 effect
```

### 3. 嵌套追踪

嵌套的副作用也能正确追踪依赖。

```typescript
effect(() => {
  console.log('outer:', state.a)
  effect(() => {
    console.log('inner:', state.b)
  })
})
```

### 4. 调度控制

可以控制副作用执行的时机和方式。

```typescript
effect(() => {
  console.log(state.count)
}, {
  scheduler(fn) {
    // 异步执行
    setTimeout(fn, 0)
  }
})
```

## 响应式系统的挑战

### 1. 对象新增属性

```typescript
const state = reactive({ count: 0 })
state.newProp = 'value'  // Vue 2 无法追踪，Vue 3 可以
```

### 2. 数组索引赋值

```typescript
const arr = reactive([1, 2, 3])
arr[0] = 10  // Vue 2 无法追踪，Vue 3 可以
```

### 3. 嵌套对象

```typescript
const state = reactive({
  nested: { value: 0 }
})
// 需要深层响应式
```

### 4. 循环引用

```typescript
const obj = reactive({})
obj.self = obj  // 需要处理循环引用
```
