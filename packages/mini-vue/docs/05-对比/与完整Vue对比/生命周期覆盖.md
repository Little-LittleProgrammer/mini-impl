# 生命周期覆盖

## 1. 完整 Vue 为什么需要这么多生命周期

生命周期不是为了让 API 看起来丰富，而是为了给不同阶段的副作用安排稳定时机。完整 Vue 至少要覆盖：

- 创建前后
- 挂载前后
- 更新前后
- 卸载前后
- 错误捕获
- 渲染调试
- keep-alive 激活和失活

## 2. 完整 Vue 的实现原理

完整 Vue 会把生命周期钩子按阶段存到组件实例上。无论这些钩子来自：

- Options API
- `onMounted` 这一类组合式 API

最终都会在组件实例初始化期间被注册进对应数组。

真正执行时，生命周期并不是直接散落在各处调用，而是嵌进 renderer 的关键节点：

- 创建实例前后触发创建相关钩子
- 首次 patch 前后触发挂载相关钩子
- 响应式副作用重新执行前后触发更新相关钩子
- 组件卸载流程中触发卸载相关钩子

错误捕获则会包裹在运行钩子、渲染函数、`setup`、调度 job 等多个执行入口外层，通过统一错误处理管线向上冒泡到最近的错误边界和应用级错误处理器。

keep-alive 的 `activated / deactivated` 更特别，它们并不等价于 mount / unmount，而是绑定在缓存子树的进出激活流程上。

## 3. 为什么教学实现只保留少量生命周期

一旦把生命周期做完整，就必须同步定义：

- 组件完整状态机
- 更新与卸载流程
- 错误处理边界
- 组合式 API 注册时机

所以教学版常常只保留最容易解释的几个挂载期钩子。

## 4. 理解重点

生命周期本质上是“把组件状态机暴露给用户副作用”的接口层，背后依赖的是完整的渲染与调度时序。
