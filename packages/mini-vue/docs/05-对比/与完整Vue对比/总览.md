# mini-vue 与完整 Vue 的区别

## 1. 先定义“完整 Vue”指什么

这里的“完整 Vue”指的是现代 Vue 3 在常规 Virtual DOM 模式下的完整能力集合，大体包含：

- 更完整的 `@vue/reactivity`
- 更完整的 `@vue/runtime-core`
- `@vue/runtime-dom`
- `@vue/compiler-core`
- `@vue/compiler-dom`
- `@vue/compiler-sfc`
- SSR / hydration 相关能力
- 更完整的应用、组件、指令、开发调试与错误处理体系

## 2. 最大差异：mini-vue 更像教学内核，不是生产框架

完整 Vue 的目标是：

- 正确性
- 性能
- 边界兼容
- API 完整性
- 跨构建链与生态协作

当前 mini-vue 的目标则更像：

- 把 Vue 3 核心思想串起来
- 能跑通最小案例
- 便于理解源码结构

- 如果你想先看结论，先读这一页。
- 如果你正在补某个能力，直接跳到对应专题，先理解完整 Vue 的设计边界，再回头看 mini-vue 缺的是什么。
- 如果你只想理解“为什么教学实现和生产框架差这么多”，优先读应用实例、组件实例、调度系统、SFC 编译链这几篇。

## 5. 一句话总结

mini-vue 和完整 Vue 的差距，不只是“少几个 API”，而是：

- 完整 Vue 需要维护一整套应用级上下文、组件实例、渲染优化、编译产物协作与工程链路
- mini-vue 只保留了最适合教学的那部分骨架
