# 模板能力差距

## 1. 完整 Vue 的模板编译器为什么会很大

模板编译器不只是把 HTML 变成 render 函数，它还要理解 Vue 自己定义的一整套模板语言，包括：

- 指令
- 参数
- 修饰符
- 插槽语法
- 组件与元素解析差异
- 表达式作用域

## 2. 完整 Vue 的实现原理

完整 Vue 的编译流程大致还是 `parse -> transform -> generate`，但每一阶段都比教学版更重。

`parse` 阶段不只是产出通用 AST，还要区分：

- 原生元素
- 组件节点
- 插槽出口
- 指令节点
- 插值表达式

`transform` 阶段是核心。它会根据不同指令做专门转换：

- `v-bind` 转成 props 合并逻辑
- `v-on` 转成事件处理器
- `v-if / v-else-if / v-else` 转成条件表达式或条件 block
- `v-for` 转成列表渲染调用
- `v-model` 转成值绑定和更新回调
- `v-slot` 转成插槽函数对象

这里还要做表达式作用域分析，避免把局部变量、插槽参数、循环别名和上下文属性混淆。

`generate` 阶段则会根据转换结果输出运行时代码，并插入辅助函数导入、优化标记、缓存表达式等内容。

## 3. 为什么“支持几个指令”不等于接近完整编译器

完整 Vue 模板能力真正复杂的地方在于：

- 各指令之间会组合
- 同一个节点可能同时带多个指令
- 指令结果要与运行时 props、slots、patchFlag 协作

因此模板能力是一个系统，不是若干零散语法糖的拼接。

## 4. 理解重点

完整 Vue 编译器的核心任务，是把模板语言里的语义约束完整翻译成运行时可执行结构，而不只是“生成一段字符串代码”。
