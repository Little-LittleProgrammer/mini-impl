# 生命周期钩子详解

## 什么是生命周期？

生命周期是组件从创建到销毁的各个阶段。Vue 提供了一系列钩子函数，允许在特定阶段执行自定义逻辑。

```typescript
import { onMounted, onUnmounted } from 'vue'

export default {
  setup() {
    onMounted(() => {
      console.log('组件已挂载')
    })

    onUnmounted(() => {
      console.log('组件已卸载')
    })
  }
}
```

## 钩子函数 API

### onBeforeMount

组件挂载之前调用。

```typescript
import { onBeforeMount } from 'vue'

onBeforeMount(() => {
  console.log('DOM 即将挂载')
})
```

### onMounted

组件挂载完成后调用。此时 DOM 已可访问。

```typescript
import { onMounted, ref } from 'vue'

const el = ref<HTMLElement | null>(null)

onMounted(() => {
  console.log('DOM 已挂载')
  console.log(el.value)  // 可以访问 DOM 元素
})
```

### onBeforeUpdate

组件更新之前调用。

```typescript
import { onBeforeUpdate } from 'vue'

onBeforeUpdate(() => {
  console.log('组件即将更新')
})
```

### onUpdated

组件更新完成后调用。

```typescript
import { onUpdated } from 'vue'

onUpdated(() => {
  console.log('组件已更新')
})
```

### onBeforeUnmount

组件卸载之前调用。

```typescript
import { onBeforeUnmount } from 'vue'

onBeforeUnmount(() => {
  console.log('组件即将卸载')
})
```

### onUnmounted

组件卸载完成后调用。用于清理副作用。

```typescript
import { onUnmounted } from 'vue'

onUnmounted(() => {
  clearInterval(timer)
  window.removeEventListener('resize', handleResize)
})
```

### onErrorCaptured

捕获后代组件错误。

```typescript
import { onErrorCaptured } from 'vue'

onErrorCaptured((err, instance, info) => {
  console.error('捕获到错误:', err)
  return false  // 阻止错误继续传播
})
```

### onRenderTracked / onRenderTriggered

调试钩子，用于追踪响应式依赖。

```typescript
import { onRenderTracked, onRenderTriggered } from 'vue'

onRenderTracked((e) => {
  console.log('依赖被追踪:', e)
})

onRenderTriggered((e) => {
  console.log('触发更新:', e)
})
```

## 实现原理

### 枚举定义

```typescript
// packages/runtime-core/src/component.ts

export const enum LifecycleHooks {
  BEFORE_CREATE = 'bc',
  CREATED = 'c',
  BEFORE_MOUNT = 'bm',
  MOUNTED = 'm',
  BEFORE_UPDATE = 'bu',
  UPDATED = 'u',
  BEFORE_UNMOUNT = 'bum',
  UNMOUNTED = 'um'
}
```

### 当前实例

Vue 使用全局变量存储当前组件实例，钩子函数通过它注册到正确的组件。

```typescript
// packages/runtime-core/src/component.ts

let currentInstance: ComponentInternalInstance | null = null

export function getCurrentInstance(): ComponentInternalInstance | null {
  return currentInstance
}

export function setCurrentInstance(instance: ComponentInternalInstance | null) {
  currentInstance = instance
}

export function unsetCurrentInstance() {
  currentInstance = null
}
```

### createHook

```typescript
// packages/runtime-core/src/apiLifecycle.ts

export function createHook<T extends Function = Function>(
  lifecycle: LifecycleHooks
): (hook: T) => void {
  return (hook: T) => {
    const currentInstance = getCurrentInstance()
    if (currentInstance) {
      injectHook(currentInstance, lifecycle, hook)
    }
  }
}
```

### injectHook

```typescript
export function injectHook(
  instance: ComponentInternalInstance,
  type: LifecycleHooks,
  hook: Function
) {
  // 获取或创建钩子数组
  const hooks = instance[type] || (instance[type] = [])

  // 包装钩子函数
  const wrappedHook = () => {
    setCurrentInstance(instance)
    try {
      hook()
    } finally {
      unsetCurrentInstance()
    }
  }

  hooks.push(wrappedHook)
}
```

### 导出的钩子函数

```typescript
export const onBeforeMount = createHook(LifecycleHooks.BEFORE_MOUNT)
export const onMounted = createHook(LifecycleHooks.MOUNTED)
export const onBeforeUpdate = createHook(LifecycleHooks.BEFORE_UPDATE)
export const onUpdated = createHook(LifecycleHooks.UPDATED)
export const onBeforeUnmount = createHook(LifecycleHooks.BEFORE_UNMOUNT)
export const onUnmounted = createHook(LifecycleHooks.UNMOUNTED)
```

## 调用钩子

### invokeArrayFns

```typescript
// packages/shared/src/general.ts

export function invokeArrayFns(fns: Function[]) {
  for (let i = 0; i < fns.length; i++) {
    fns[i]()
  }
}
```

### 在挂载时调用

```typescript
const setupRenderEffect = (instance, vnode, container, anchor) => {
  const componentUpdateFn = () => {
    if (!instance.isMounted) {
      const { bm, m } = instance

      // beforeMount
      if (bm) {
        invokeArrayFns(bm)
      }

      // 渲染...

      // mounted（异步）
      if (m) {
        queuePostFlushCb(() => {
          invokeArrayFns(m)
        })
      }
    }
  }
}
```

### 在更新时调用

```typescript
// 更新
const { bu, u } = instance

// beforeUpdate
if (bu) {
  invokeArrayFns(bu)
}

// 重新渲染...

// updated（异步）
if (u) {
  queuePostFlushCb(() => {
    invokeArrayFns(u)
  })
}
```

### 在卸载时调用

```typescript
const unmount = (vnode: VNode) => {
  const { shapeFlag } = vnode

  if (shapeFlag & ShapeFlags.COMPONENT) {
    unmountComponent(vnode.component!)
  }
}

const unmountComponent = (instance: ComponentInternalInstance) => {
  const { bum, um } = instance

  // beforeUnmount
  if (bum) {
    invokeArrayFns(bum)
  }

  // 卸载子树...

  // unmounted（异步）
  if (um) {
    queuePostFlushCb(() => {
      invokeArrayFns(um)
    })
  }
}
```

## 异步执行

`mounted`、`updated`、`unmounted` 钩子通过 `queuePostFlushCb` 异步执行，确保在所有 DOM 更新完成后执行。

```typescript
import { queuePostFlushCb } from './scheduler'

// mounted 钩子
if (m) {
  queuePostFlushCb(() => {
    invokeArrayFns(m)
  })
}
```

## 钩子的执行顺序

### 父子组件

```
父 beforeMount
  子 beforeMount
  子 mounted
父 mounted
```

```
父 beforeUpdate
  子 beforeUpdate
  子 updated
父 updated
```

```
父 beforeUnmount
  子 beforeUnmount
  子 unmounted
父 unmounted
```

### 实现

```typescript
// 父组件先注册，子组件后注册
// 但异步执行时是先执行子组件的钩子

// 在组件挂载时
if (m) {
  queuePostFlushCb(() => {
    invokeArrayFns(m)
  })
}
```

因为 `queuePostFlushCb` 是按注册顺序执行的，而子组件的钩子是在父组件的渲染过程中注册的，所以会先执行子组件的钩子。

## 与 Vue 2 的区别

| Vue 2 | Vue 3 |
|-------|-------|
| beforeCreate | setup() |
| created | setup() |
| beforeMount | onBeforeMount |
| mounted | onMounted |
| beforeUpdate | onBeforeUpdate |
| updated | onUpdated |
| beforeDestroy | onBeforeUnmount |
| destroyed | onUnmounted |

## 最佳实践

### 1. 清理副作用

```typescript
import { onUnmounted, ref } from 'vue'

const timer = ref<number | null>(null)

onMounted(() => {
  timer.value = setInterval(() => {
    console.log('tick')
  }, 1000)
})

onUnmounted(() => {
  if (timer.value) {
    clearInterval(timer.value)
  }
})
```

### 2. 访问 DOM

```typescript
import { onMounted, ref } from 'vue'

const el = ref<HTMLElement | null>(null)

onMounted(() => {
  // 此时 DOM 已挂载，可以安全访问
  el.value?.focus()
})

return { el }
```

### 3. 避免在 updated 中修改状态

```typescript
// 错误：可能导致无限循环
onUpdated(() => {
  count.value++  // 触发更新 → 调用 onUpdated → 修改 count → ...
})

// 正确：只在特定条件下修改
onUpdated(() => {
  if (needsUpdate.value) {
    needsUpdate.value = false
    // ...
  }
})
```

### 4. 使用 watchEffect 自动清理

```typescript
import { watchEffect, onUnmounted } from 'vue'

// watchEffect 返回停止函数
const stop = watchEffect(() => {
  // 自动追踪依赖
})

onUnmounted(stop)
```

## 总结

生命周期钩子的核心要点：

| 钩子 | 时机 | 用途 |
|------|------|------|
| onBeforeMount | 挂载前 | 最后的准备工作 |
| onMounted | 挂载后 | 访问 DOM、启动定时器 |
| onBeforeUpdate | 更新前 | 获取更新前的 DOM 状态 |
| onUpdated | 更新后 | 响应 DOM 变化 |
| onBeforeUnmount | 卸载前 | 清理前的准备工作 |
| onUnmounted | 卸载后 | 清理副作用 |
