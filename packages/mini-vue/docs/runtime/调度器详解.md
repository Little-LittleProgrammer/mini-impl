# 调度器详解

## 什么是调度器？

调度器负责管理副作用的执行时机，实现批量更新和异步更新。它确保多次状态变更只触发一次 DOM 更新，提高性能。

```typescript
const state = reactive({ count: 0 })

// 没有 scheduler
state.count++
state.count++
state.count++
// 渲染 3 次

// 有 scheduler
state.count++
state.count++
state.count++
// 只渲染 1 次（合并更新）
```

## 任务队列

### 任务定义

```typescript
// packages/runtime-core/src/scheduler.ts

export interface SchedulerJob {
  (): void
  id?: number
  pre?: boolean
  active?: boolean
  computed?: boolean
}
```

### 任务队列

```typescript
const queue: SchedulerJob[] = []
let flushIndex = 0
let isFlushing = false
let isFlushPending = false
```

## queueJob

将任务添加到队列，自动去重。

```typescript
export function queueJob(job: SchedulerJob) {
  // 检查是否已在队列中
  if (!queue.includes(job, flushIndex)) {
    queue.push(job)
    queueFlush()
  }
}
```

## queueFlush

触发队列刷新。

```typescript
function queueFlush() {
  // 如果没有在刷新，也没有待刷新
  if (!isFlushing && !isFlushPending) {
    isFlushPending = true
    // 使用 Promise 异步执行
    currentFlushPromise = resolvedPromise.then(flushJobs)
  }
}
```

## flushJobs

执行队列中的所有任务。

```typescript
function flushJobs() {
  isFlushPending = false
  isFlushing = true

  // 按优先级排序
  // pre 任务先执行
  queue.sort((a, b) => getId(a) - getId(b))

  try {
    for (flushIndex = 0; flushIndex < queue.length; flushIndex++) {
      const job = queue[flushIndex]
      if (job && job.active !== false) {
        job()
      }
    }
  } finally {
    // 重置
    flushIndex = 0
    queue.length = 0
    isFlushing = false
    currentFlushPromise = null

    // 执行后置回调
    if (pendingPostFlushCbs.length) {
      flushPostFlushCbs()
    }
  }
}
```

## Pre Flush 回调

在组件更新前执行的回调。

```typescript
const pendingPreFlushCbs: SchedulerJob[] = []

export function queuePreFlushCb(cb: SchedulerJob) {
  queueJob(cb)
}

// flushJobs 开始时先执行 pre 回调
function flushPreFlushCbs() {
  for (let i = 0; i < pendingPreFlushCbs.length; i++) {
    pendingPreFlushCbs[i]()
  }
  pendingPreFlushCbs.length = 0
}
```

## Post Flush 回调

在 DOM 更新后执行的回调。

```typescript
const pendingPostFlushCbs: SchedulerJob[] = []

export function queuePostFlushCb(cb: SchedulerJob | SchedulerJob[]) {
  if (Array.isArray(cb)) {
    pendingPostFlushCbs.push(...cb)
  } else {
    pendingPostFlushCbs.push(cb)
  }
  queueFlush()
}

function flushPostFlushCbs() {
  if (pendingPostFlushCbs.length) {
    const cbs = [...new Set(pendingPostFlushCbs)]
    pendingPostFlushCbs.length = 0

    cbs.sort((a, b) => getId(a) - getId(b))

    for (let i = 0; i < cbs.length; i++) {
      cbs[i]()
    }
  }
}
```

## nextTick

等待 DOM 更新完成。

```typescript
export function nextTick<T = void>(
  fn?: () => T
): Promise<T> {
  const p = currentFlushPromise || resolvedPromise
  return fn ? p.then(fn) : p
}
```

### 使用示例

```typescript
import { nextTick, ref } from 'vue'

const count = ref(0)

count.value++
// DOM 还未更新

await nextTick()
// DOM 已更新

// 或者使用回调
nextTick(() => {
  // DOM 已更新
})
```

## 调度器与组件更新

### 组件更新函数

```typescript
const setupRenderEffect = (instance, vnode, container, anchor) => {
  const componentUpdateFn = () => {
    // 渲染逻辑...
  }

  // 创建响应式 effect，使用 scheduler
  const effect = new ReactiveEffect(
    componentUpdateFn,
    () => queueJob(update)  // 调度器
  )

  const update = instance.update = effect.run.bind(effect)

  // 首次执行
  update()
}
```

### 工作流程

```
state.count++
    │
    ▼
trigger(state, 'count')
    │
    ▼
执行 effect.scheduler
    │
    ▼
queueJob(update)
    │
    ▼
添加到队列（去重）
    │
    ▼
Promise.then(flushJobs)
    │
    ▼
异步执行所有任务
```

## 为什么使用 Promise？

### 微任务队列

`Promise.then` 将任务放入微任务队列，在当前同步代码执行完后立即执行。

```typescript
state.count = 1
state.count = 2
state.count = 3

// 同步代码执行完毕后
// 微任务队列: [flushJobs]
// flushJobs 执行一次，渲染 count = 3
```

### 对比其他方案

| 方案 | 执行时机 | 适用场景 |
|------|----------|----------|
| 同步 | 立即 | 简单场景 |
| setTimeout | 宏任务 | 延迟执行 |
| Promise | 微任务 | DOM 更新 |
| requestAnimationFrame | 帧刷新 | 动画 |

## 任务优先级

```typescript
// 任务按 id 排序
// id 越小，越先执行

// 组件 id
let uid = 0

function getId(job: SchedulerJob): number {
  return job.id == null ? Infinity : job.id
}

// 排序
queue.sort((a, b) => getId(a) - getId(b))
```

### 父子组件顺序

父组件先创建，id 更小，所以先执行父组件的更新。但在 mounted 钩子中，子组件先注册，所以先执行子组件的钩子。

## 无限循环检测

```typescript
// 检测递归更新
let checkRecursiveUpdates = true
const RECURSION_LIMIT = 100

function flushJobs() {
  // ...
  for (flushIndex = 0; flushIndex < queue.length; flushIndex++) {
    const job = queue[flushIndex]
    if (checkRecursiveUpdates && job.__dirty) {
      // 检测到递归更新
      if (job.__runs > RECURSION_LIMIT) {
        console.warn('Maximum recursive updates exceeded')
      }
    }
    job()
  }
}
```

## 实际应用

### watch 的调度

```typescript
const count = ref(0)

watch(count, (newVal) => {
  console.log('count changed:', newVal)
}, {
  flush: 'pre'  // 默认，组件更新前
  // flush: 'post'  // DOM 更新后
  // flush: 'sync'  // 同步执行
})
```

```typescript
// 实现
function doWatch(source, cb, { flush }) {
  let scheduler: EffectScheduler

  if (flush === 'sync') {
    scheduler = job
  } else if (flush === 'post') {
    scheduler = () => queuePostFlushCb(job)
  } else {
    scheduler = () => queuePreFlushCb(job)
  }

  const effect = new ReactiveEffect(getter, scheduler)
}
```

### computed 的调度

computed 使用调度器来标记脏状态：

```typescript
class ComputedRefImpl {
  constructor(getter) {
    this.effect = new ReactiveEffect(getter, () => {
      if (!this._dirty) {
        this._dirty = true
        triggerRefValue(this)
      }
    })
  }
}
```

## 总结

调度器的核心功能：

| 功能 | 说明 |
|------|------|
| 批量更新 | 合并多次状态变更 |
| 异步执行 | 使用 Promise 微任务 |
| 去重 | 避免重复执行 |
| 优先级 | 按 id 排序执行 |
| nextTick | 等待 DOM 更新 |
